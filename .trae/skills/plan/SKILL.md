# 设计技术方案技能

## 核心能力 

作为项目的首席架构师，你需要根据 `spec.md` 为项目生成一份详细的技术实现方案 `plan.md` 。

## 触发条件

当用户的请求包含以下关键词时，你应该优先考虑激活本技能：
- “制定技术方案”
- “设计技术方案”

## 执行步骤 

当你决定使用本技能时，必须严格遵循以下步骤：

1. **读取 `spec.md` 文件:** 首先，你必须读取用户提供的 `spec.md` 文件，确保理解其中的内容
2. **读取 `constitution.md` 文件:** 首先，你必须读取并遵守项目根目录下的 `constitution.md`原则，检查并确认本技术方案符合所有条款，特别是：
   - TDD
   - 包内聚
   - 错误处理
3. **根据 `spec.md` 创作 `plan.md`:** 基于 `spec.md` 中的内容，创作一份详细的技术实现方案 `plan.md` 。文件内容必须包含以下内容：
   - **技术上下文总结:** 明确上述技术选型
   - **“合宪性”审查:** 逐条对照`constitution.md`的原则，检查并确认本技术方案符合所有条款（特别是包内聚、错误处理、TDD）。
   - **项目结构细化:** 
   - **核心数据结构:** 
   - **接口设计:** 
4. **维护好 `plan.md`:** 将生成的`plan.md`文件，放在用户提供的`spec.md`所在目录下
