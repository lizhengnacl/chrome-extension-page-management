# 任务拆解技能

## 核心能力

作为一名技术组长。将用户提供的 `plan.md` 文件，分解成一个详尽的、原子化的、有依赖关系的、可被AI直接执行的任务列表。

## 触发条件

当用户的请求包含以下关键词时，你应该优先考虑激活本技能：
- “拆解技术方案”
- “任务分解”

## 执行步骤 

1. **读取 `plan.md` 文件:** 首先，你必须读取用户提供的 `plan.md` 文件，确保理解其中的内容
2. **读取 `constitution.md` 文件:** 你必须读取并遵守项目根目录下的 `constitution.md`原则，检查并确认本技术方案符合所有条款，特别是：
   - TDD
   - 包内聚
   - 错误处理
3. **根据 `plan.md` 创作 `tasks.md`:** 基于 `plan.md` 中的内容，创作一份详细的任务列表 `tasks.md` 。关键要求：
   - **任务粒度：** 每个任务应该只涉及一个主要文件的修改或创建一个新文件。不要出现“实现所有功能”这种大任务。
   - **TDD强制：** 根据`constitution.md`的“测试先行铁律”，采用表格驱动测试，**必须**先生成测试任务，后生成实现任务。
   - **并行标记：** 对于没有依赖关系的任务，请标记 `[P]`。
   - **阶段划分：** 明确执行阶段划分
4. **维护好 `tasks.md` :** 将生成的`tasks.md`文件，放在用户提供的`plan.md`所在目录下
