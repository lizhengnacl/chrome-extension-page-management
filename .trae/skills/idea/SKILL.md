# 技能自动化执行引擎技能

## 核心能力 

将模糊的创意或需求，通过一个固定的、四阶段标准化流程，转化为可直接落地执行的具体方案或代码。本技能作为总控入口，严格按序调用项目内四个子技能，确保从概念到成品的转换过程清晰、可靠、可复现

## 触发条件

当用户输入一个项目构想、功能需求或待解决的问题描述，并以指令 /idea触发时，本流程启动

## 执行步骤

本技能将作为协调者，严格、顺序、自动化地执行以下四个步骤，前序步骤的输出将作为后续步骤的唯一输入，构成一个完整的工作流水线。

阶段一：定义规格 (Call /spec)
- 输入：用户的初始想法描述。
- 动作：调用 /spec技能，对想法进行分析、澄清和结构化定义，明确目标、范围、功能点与约束条件。
- 交付物：生成一份作为项目基线的 spec.md​ 文档。仅当此文档生成后，进入下一阶段。

阶段二：制定计划 (Call /plan)
- 输入：上一阶段生成的 spec.md​ 文档。
- 动作：调用 /plan技能，依据确认的规格，设计实现路径，将项目分解为逻辑上的主要阶段或模块，并规划其执行顺序与依赖。
- 交付物：生成一份指导性的 plan.md​ 文档。仅当此文档生成后，进入下一阶段。

阶段三：分解任务 (Call /tasks)
- 输入：上一阶段生成的 plan.md​ 文档。
- 动作：调用 /tasks技能，将计划中的每个阶段进一步拆解为最小可执行、可验证的原子任务项，形成待办清单。
- 交付物：生成一份可操作的 tasks.md​ 任务列表文档。仅当此文档生成后，进入最终阶段。

阶段四：生成执行件 (Call /exec)
- 输入：上一阶段生成的 tasks.md​ 文档。
- 动作：调用 /exec技能，针对当前任务列表的优先级，生成具体的、可立即运行或实施的代码、命令、脚本或详细操作步骤。
- 交付物：输出与任务对应的具体可执行成果。


流程规则：本技能 (/idea) 将强制执行上述串行顺序，用户无需手动调用中间子技能。除非流程报错或用户明确中断，否则将自动运行直至完成全部四个阶段。
