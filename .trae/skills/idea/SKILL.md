# 技能自动化执行引擎技能

## 核心能力 

将一个初步的想法（idea）自动转化为可具体执行的任务列表，并生成执行代码。本引擎通过四个严格串行的阶段（spec -> plan -> tasks -> exec）将抽象概念逐步具体化、可操作化

## 触发条件

当用户提出一个具体的、需要被分解和实现的项目想法或目标时
- "我有一个想法"
- "/idea "

## 执行步骤

执行步骤：
0. 初始化阶段：
    ◦  导入项目下的所有技能 .trae/skills/
1.  spec阶段 (规格定义)：
    ◦   输入：用户提供的初始想法描述。
    ◦   动作：调用项目下的 /spec 执行
3.  plan阶段 (计划制定)：
    ◦   输入：上一步生成的 spec.md 文档。
    ◦   动作：调用项目下的 /plan 执行
4.  tasks阶段 (任务分解)：
    ◦   输入：上一步生成的 plan.md 文档。
    ◦   动作：调用项目下的 /tasks 执行
5.  exec阶段 (执行生成)：
    ◦   输入：上一步生成的 tasks.md 文档。
    ◦   动作：调用项目下的 /exec 执行

流程纪律：必须严格按照 spec -> plan -> tasks -> exec 的顺序执行，每个阶段的输出必须作为下一阶段唯一的输入依据。除非用户明确要求修改或跳转，否则不得更改此顺序或跳过中间步骤。
