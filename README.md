# Chrome 页面管理插件

一个轻量级的Chrome浏览器插件，帮助您快速保存、分类和管理您的网页。

## 功能特性

- 📄 **页面管理** - 添加、编辑、删除和打开保存的页面
- 📁 **分组功能** - 将页面组织到不同的分组中
- 🏷️ **标签系统** - 为页面添加标签，便于筛选
- 🔍 **智能搜索** - 按标题、URL、标签和备注搜索页面
- 📊 **多种排序** - 按最近访问、创建时间、访问次数和标题排序
- ⚡ **快捷操作** - 右键菜单、快捷键和一键添加当前页面
- 💾 **数据同步** - 使用Chrome Sync在设备间同步数据
- 📤 **导入导出** - 备份和恢复您的数据
- ♻️ **撤销删除** - 误删页面可以快速恢复

## 安装说明

### 开发模式安装

1. 克隆或下载此项目到本地
2. 打开Chrome浏览器，访问 `chrome://extensions/`
3. 在右上角开启"开发者模式"
4. 点击"加载已解压的扩展程序"
5. 选择项目文件夹
6. 插件安装成功！

### 使用说明

#### 基本操作

1. **添加页面**
   - 点击插件图标打开popup
   - 点击"添加页面"按钮手动添加
   - 点击"添加当前页"快速保存当前浏览的页面
   - 在页面上右键，选择"添加到页面管理"
   - 使用快捷键 `Ctrl+Shift+S` (Mac: `Cmd+Shift+S`)

2. **打开页面**
   - 点击页面标题即可在新标签页中打开
   - 打开页面会自动记录访问次数

3. **编辑页面**
   - 点击页面卡片上的✏️图标
   - 修改URL、标题、分组、标签或备注
   - 点击"保存"按钮

4. **删除页面**
   - 点击页面卡片上的🗑️图标
   - 页面删除后会显示撤销按钮
   - 5秒内可点击"撤销"恢复页面

#### 分组管理

1. **创建分组**
   - 点击"新建分组"按钮
   - 输入分组名称
   - 点击"创建"

2. **使用分组**
   - 添加或编辑页面时选择分组
   - 点击顶部分组按钮筛选页面

#### 标签系统

1. **添加标签**
   - 添加或编辑页面时输入标签
   - 多个标签用逗号分隔
   - 例如：`工作, 学习, 参考`

2. **筛选标签**
   - 点击标签云中的标签筛选页面
   - 再次点击取消筛选

#### 搜索和排序

1. **搜索页面**
   - 在搜索框中输入关键词
   - 会实时匹配标题、URL、标签和备注

2. **排序页面**
   - 选择排序方式：最近访问、创建时间、访问次数、标题
   - 点击⬇️/⬆️按钮切换升序/降序

#### 数据管理

1. **导出数据**
   - 点击📤按钮
   - 自动下载JSON格式的备份文件

2. **导入数据**
   - 点击📥按钮
   - 选择之前导出的JSON文件
   - 数据会自动恢复

3. **数据同步**
   - 数据自动通过Chrome Sync同步
   - 在登录同一Google账号的设备间共享

#### 快捷键

- `Ctrl+Shift+P` (Mac: `Cmd+Shift+P`) - 打开插件popup
- `Ctrl+Shift+S` (Mac: `Cmd+Shift+S`) - 快速添加当前页面

## 调试说明

### 查看日志

1. **Popup日志**
   - 右键点击插件图标
   - 选择"检查弹出内容"
   - 在Console标签中查看日志

2. **后台脚本日志**
   - 访问 `chrome://extensions/`
   - 找到本插件，点击"Service Worker"
   - 在Console标签中查看日志

### 调试步骤

1. **修改代码后**
   - 访问 `chrome://extensions/`
   - 点击插件的刷新按钮🔄
   - 重新打开popup测试

2. **查看存储数据**
   - 打开Popup的开发者工具
   - 在Console中输入：
     ```javascript
     chrome.storage.sync.get(null, (data) => console.log(data))
     ```

3. **清空存储数据**
   - 在Console中输入：
     ```javascript
     chrome.storage.sync.clear()
     ```

## 技术栈

- **原生JavaScript** - 不使用任何前端框架
- **Chrome Extension Manifest V3** - 最新的插件规范
- **Chrome Storage API** - 数据存储和同步
- **ES Modules** - 模块化代码组织

## 开发原则

本项目严格遵循以下开发原则：

- 🧪 **TDD优先** - 测试先行，红灯-绿灯-重构
- 🎯 **简单性原则** - YAGNI，避免过度设计
- 📦 **单一职责** - 每个模块只负责一个功能
- 🔒 **无第三方依赖** - 只使用Chrome原生API
